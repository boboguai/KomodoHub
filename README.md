# NiuYang Conservation (coursework prototype)

Next.js + Prisma (SQLite) demo inspired by the **Komodo Hub** case study, extended for a **worldwide** endangered species database, **daily points**, **rewards**, and a **conservation books library**.

**Organisation name:** *NiuYang Endangered Wildlife Protection Organization*.

## Prerequisites

- Node.js 18+
- npm

## Desktop (Tauri)

The `src-tauri/` crate wraps the site in a native window. **Development** uses the same Next dev server (`npm run dev`) and loads `http://localhost:3000`. **Release** bundles the [Next standalone](https://nextjs.org/docs/app/building-your-application/deploying#nodejs-server) output into `src-tauri/server_bundle` (generated by `npm run build:tauri`; do not edit by hand).

```bash
cd komodo-hub
npm install
npm run tauri:dev      # Next dev + desktop window
npm run tauri:build    # production installer (needs Rust toolchain + Windows: WebView2)
```

**Runtime Node.js:** the packaged app starts `node server.js` from `server_bundle`. By default it uses **`node` on the system PATH**. To ship Node inside the app, extract the official Windows **x64** Node zip and place `node.exe` at `src-tauri/resources/node/node.exe`, then add that path under `bundle.resources` in `tauri.conf.json` (same pattern as `server_bundle/`).

### Installer size (rough, Windows x64)

These are **order-of-magnitude** numbers; your exact `.nsis` size depends on compression and what you embed.

| Component | Typical contribution |
|-----------|----------------------|
| Tauri shell + NSIS packaging | ~5–20 MB |
| `server_bundle` (Next standalone + traced `node_modules`, Prisma, `public`, `.next/static`) | ~80–200 MB |
| Portable Node runtime (optional `node.exe` + deps) | ~35–55 MB |
| WebView2 | Usually **0 MB** in the installer on Win10/11 (Evergreen runtime already on the machine). An offline bootstrapper that installs WebView2 can add **~50–80 MB**. |

**Overall:** expect about **100–220 MB** for a typical build with bundled `server_bundle` and system Node; **+35–50 MB** if you embed Node; **+50–80 MB** if you also bundle WebView2 for offline machines.

## Setup

**Windows:** double-click the parent-folder launcher batch file, or run `start.bat` inside this folder (`npm install`, `prisma db push`, `db:seed`, `npm run dev`).

Manual:

```bash
cd komodo-hub
npm install
cp .env.example .env   # or keep generated .env
npx prisma generate
npx prisma db push --skip-generate
npm run db:seed
npm run dev
```

Open `http://localhost:3000`.

`package.json` intentionally has **no `postinstall` Prisma generate** (Windows often hits **EPERM** when `npm install` renames `query_engine-windows.dll.node` while another process holds it). `start.bat` runs `prisma generate` after install with retries. If it still fails: stop all **node.exe** using this folder, pause antivirus on `komodo-hub`, or move the repo off a OneDrive-synced path, then run `npx prisma generate` again.

All default species/library/reward images are local files under `public/images` (no real-time download).

## Features (high level)

- Guest browsing of species + library books; login with optional **role check** (student / teacher / school admin / foundation admin / community).
- **Profile:** display name, nickname, gender, bio, emoji + image avatar URL, theme.
- **Daily:** check-in (+5) and missions (reading / share / explore) for points.
- **Rewards:** redeem cups / backpack / pens with **user-chosen species artwork** (prototype: pending fulfilment).
- **School library page:** books on species lists, threats, protection actions; filter `?school=ujung-raya`.
